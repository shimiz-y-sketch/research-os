# 🗺️ Project Overview  
## iTenFs 開発プロジェクト

---

# 🎯 プロジェクトの目的

本プロジェクトの目的は、  
**iTenFs アプリケーションの編集データ管理機能を拡張し、  
非破壊編集・表示・エクスポートの一貫したデータフローを構築すること**です。

主な目標は次の通りです。

- 🖼️ **画像編集データの非破壊管理**
- 💾 **編集データの永続化**
- 📊 **一覧画面・サムネイルへの反映**
- 📦 **エクスポート機能への統合**

これにより、アプリケーションの次の価値を高めます。

- データ整合性
- 編集履歴の保持
- 表示性能
- 外部連携能力

---

# 🧭 開発フェーズ構成

本プロジェクトは以下のフェーズで進行します。

```
PR2
↓
PR3
↓
PR4
↓
PR5
```

各フェーズは、**機能単位で段階的に拡張する設計**になっています。

---

# 📦 PR2  
## UI層の編集データ対応

目的

- 編集データを UI 層で扱えるようにする
- ImageEditorView の API を整理する
- フォーム画面から編集データを取得できるようにする

主な内容

- ImageEditorView API変更
- PhotoFormView 編集データ保持
- SpotFormView 編集データ取得
- GroupFormView ベース画像編集対応

このフェーズでは

**UI レイヤーで編集データを扱える基盤**を作ります。

---

# 💾 PR3  
## データフローと永続化

目的

- 編集データをデータモデルへ保存する
- 既存データとの互換性を保つ
- 読み込み・復元フローを整備する

主な内容

- PhotoImage への編集データ保存
- PhotoOnMap / Group への保存
- データマイグレーション
- エラーハンドリング
- 単体テスト

このフェーズでは

**編集データの永続化基盤**を確立します。

---

# 🖥️ PR4  
## 表示対応（サムネイル・一覧表示）

目的

- 編集済み画像を UI 表示へ反映する
- サムネイル生成を行う
- 表示パフォーマンスを最適化する

主な内容

- サムネイル生成ロジック
- 一覧画面表示対応
- キャッシュ機構
- 表示パフォーマンス最適化

---

目的

# 🧠 プロジェクト管理方式

本プロジェクトでは、  
**AI と Git に適したプロジェクト管理構造**を採用しています。

従来の Excel ガント管理ではなく、  
以下の構造で管理します。

```
YAML
 ↓
Mermaid
 ↓
Markdown
```

---

# 📊 管理レイヤー

|レイヤー|役割|
|---|---|
|WBS|作業構造|
|Schedule|予定・実績|
|Progress|進捗ログ|
|Mermaid|ガント表示|
|Markdown|背景・判断|

---

# 🤖 AI活用方針

本プロジェクトでは AI を

**実装補助だけでなく、  
プロジェクト管理補助にも活用します。**

AI に任せる役割

- 進捗更新
- ガント更新
- ステータスレポート生成
- リスク整理
- 遅延分析

人間が握る役割

- 優先順位
- スコープ
- 完了条件
- 技術判断
- リスク判断

---

# 🧭 この管理方式の一言まとめ

> **YAML を真実のソースとし、Mermaid は可視化、Markdown は意味を持つ**

この分離により

- AIが扱いやすい
- Gitで履歴が追いやすい
- 人間が理解しやすい

プロジェクト管理構造を維持します。
