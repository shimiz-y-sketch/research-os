# 🧠 Research OS
## Operating System for the R&D Team

---

# 🌍 Overview

**Research OS** とは、

> **研究開発チームを動かすための Operating System（基盤）**

である。

これは単なる

- 📚 Knowledge Base  
- 📘 Engineering Handbook  
- 🗂 Documentation Repository  

ではない。

Research OS はそれらより **一段上のレイヤー**に存在する。

Research OS は

- 🧭 **思想 (Philosophy)**
- 🧠 **研究戦略 (Research Strategy)**
- ⚙️ **運用ルール (Operational Rules)**
- 📚 **知識体系 (Knowledge)**
- 📜 **意思決定履歴 (Decision Records)**

を統合した

> **研究開発組織の Operating System**

である。

---

# 🧠 なぜ Research OS が必要なのか

AI時代の研究開発では、従来の開発組織とは大きく前提が変わっている。

---

## 1️⃣ 研究開発の複雑性の爆発

現代の研究開発は

- AI
- 数理
- ソフトウェア
- インフラ
- データ

が複雑に絡み合う。

例えば

```
Photogrammetry
        ↓
3D Reconstruction
        ↓
Physics Simulation
        ↓
iPINNs
        ↓
Infrastructure AI
```

さらに

```
RAG
↓
Domain Knowledge
↓
Engineering Support
```

のような補助系AIも存在する。

つまり

> **研究テーマは単体では存在しない**

---

## 2️⃣ 知識の寿命が短い

AI時代では

- ツール
- モデル
- ライブラリ
- 開発プロセス

が高速に変化する。

例：

```
2022
PyTorch + CNN

2023
Transformer

2024
Foundation Model

2025
Agentic AI
```

つまり

> 静的なドキュメントでは組織は維持できない

---

## 3️⃣ AIが開発プロセスに参加する

従来

```
Human → Code
```

だった。

しかし現在は

```
Human → AI → Code
```

である。

さらに

```
Human
  ↓
AI planning
  ↓
AI coding
  ↓
Human review
```

となる。

つまり

> **ドキュメントは AI が読む前提で書かれる必要がある**

---

## 4️⃣ 研究開発は「OS」に近い

研究チームを支えるものは

- 思想
- 方針
- ルール
- 知識
- 判断履歴

である。

これは

**コンピュータにおける OS に近い。**

# 🧭 Research OS の階層構造

Research OS は以下の階層で構成される。

```
Research OS
│
├── Philosophy
│
├── Strategy
│
├── Operations
│
├── Knowledge
│
└── Decisions
```

---

# 🧠 1. Philosophy（思想）

最上位レイヤー。

ここでは

- 世界観
- 原則
- 判断基準

を定義する。

例：

### Determinism

再現性を重視する。

### Reproducibility

研究結果は再現可能であるべき。

### Docs as Code

ドキュメントはコードとして扱う。

### AI Co-development

AIと共同で開発する。

---

# 🧭 2. Strategy（戦略）

ここでは

- 技術ポートフォリオ
- 研究方向
- 技術獲得ロードマップ

を定義する。

例：

```
Research Portfolio

iTenFs
Photogrammetry
iPINNs
RAG
```

---

# ⚙️ 3. Operations（運用）

ここでは

- Gitルール
- PR作法
- コミット粒度
- 会議運営

などを定義する。

例：

```
development/
    git-strategy.md
    pr-rules.md
    commit-granularity.md
```

---

# 📚 4. Knowledge（知識）

ここには

- 技術資料
- 研究ノート
- 設計思想
- アーキテクチャ

を置く。

---

# 📜 5. Decisions（意思決定履歴）

ここには

**ADR (Architecture Decision Record)** を保存する。

例

```
ADR-001
Use GitHub + Markdown + MkDocs

ADR-002
Docs-as-Code policy
```
